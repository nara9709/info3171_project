Inline Notifications

Inline notifications are used to notify a user of the status of an action during a task flow. It is recommended that inline notifications are shown at the top of the main content area.

Examples

Good! This is working out.
This is some general information. You should know about this.
There is a problem! Someting terrible happened View Details.

Code Snippet

Note

Inline notifications are categorized into four types based on severity:

Success: indicates that an action processed successfully or that there are no issues. The success notification uses green color.
Warning or Info: provides general information or indicates that an action failed at this time. Use the yellow background color.
Error:indicates that an action failed completely. Use the error class for the red background color.

*Link (optional): Show a link to allow the user to navigate to details of the message.

Note

If the table width is larger than the screen, the table is scrolled horizontally. Also, if the table height is greater than 300px, it will scroll vertically. Thread tag is fixed, so it doesn't move when you scroll down.

CSS information of the table is specified in the properties of div, table, thread, tr and, th tag.

If you do not want to use both horizontal and vertical scrolling, delete the overflow-x:auto property of div style and delete the class overflow property of the same div tag as well.
